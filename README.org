#+title: aws-logs.el repository

This repository contains four related Emacs Lisp packages:

- ~aws-logs~: transient-driven CloudWatch Logs tooling (tail + Logs Insights).
- ~kube-logs~: transient-driven Kubernetes logs tooling (pods/deployments via ~kubectl~).
- ~kafka-logs~: transient-driven Kafka message viewer (via ~kcat~).
- ~json-log-viewer~: generic foldable JSON log viewer used by ~aws-logs~ and usable standalone.

* Package: aws-logs

~aws-logs~ provides an Emacs transient UI for AWS CloudWatch Logs tailing and Logs Insights queries.

** Features

- Tail CloudWatch logs with profile/region/log-group controls.
- Run Logs Insights queries and view foldable, formatted results.
- Configure summary formatting (timestamp/level/message + optional extra fields).
- Save/load reusable query files from disk.
- Define option presets and apply them from transient.
- Non-blocking runtime paths for EXWM-friendly usage:
  - Logs Insights run/refresh is asynchronous.
  - ECS one-shot tail fetch is asynchronous.
  - ECS follow mode batches UI pushes to reduce redraw churn.

** Files

- ~aws-logs.el~: transient UI, session state, presets, saved-query management.
- ~aws-logs-query.el~: Logs Insights query major mode + popup editor.
- ~aws-logs-insights.el~: asynchronous Insights execution and rendering.
- ~aws-logs-tail.el~: tail execution (comint + ECS/json-log-viewer integration).

** Installation

Put these files on your ~load-path~:

- ~aws-logs.el~
- ~aws-logs-query.el~
- ~aws-logs-insights.el~
- ~aws-logs-tail.el~
- ~json-log-viewer.el~

** use-package example

#+begin_src emacs-lisp
(use-package aws-logs
  :load-path "~/dev/emacs/aws-logs.el"
  :commands (aws-logs)
  :custom
  (aws-logs-cli "aws")
  (aws-logs-region "us-east-1")
  (aws-logs-default-profile "engineering")
  (aws-logs-default-log-group "/aws/ecs/payments-api")
  (aws-logs-default-follow t)
  (aws-logs-default-since "20m")
  (aws-logs-insights-saved-queries-directory
   (expand-file-name "aws-logs/insights-queries/" user-emacs-directory))
  (aws-logs-default-summary-level-field "event.level")
  (aws-logs-default-summary-message-field "event.message")
  (aws-logs-default-summary-extra-fields '("event.service" "event.logger"))
  ;; ECS follow batching (optional tuning for high-volume streams):
  (aws-logs-tail-ecs-batch-flush-interval 0.08)
  (aws-logs-tail-ecs-batch-max-lines 200)
  :config
  (aws-logs-make-preset
   "prod-errors"
   :profile "prod-observability"
   :log-group "/aws/lambda/orders-prod-handler"
   :since "45m"
   :follow nil
   :query "fields @timestamp, @message | filter event.level = \"ERROR\" | sort @timestamp desc"
   :summary-timestamp-field "@timestamp"
   :summary-level-field "event.level"
   :summary-message-field "event.message"
   :summary-extra-fields '("event.service" "event.logger")))
#+end_src

** Presets

~aws-logs-make-preset~ stores named presets in ~aws-logs-presets~.

Supported keys:

- ~:log-group~
- ~:since~
- ~:custom-time-range~ (cons cell: ~(FROM . TO)~)
- ~:follow~
- ~:ecs~
- ~:profile~
- ~:query~
- ~:summary-timestamp-field~
- ~:summary-level-field~
- ~:summary-message-field~
- ~:summary-extra-fields~

Apply behavior:

- Key missing: keep current value.
- Key present with ~nil~: clear value.
- Key present with non-~nil~: set value.

** Saved queries

Saved queries are file-based. Use ~aws-logs-insights-saved-queries-directory~ to choose the directory.

From transient:

- ~q~ opens query submenu.
- ~e~ edits active query.
- ~s~ saves active query to file.
- ~l~ loads a saved query file.

** Usage

Run ~M-x aws-logs~.

Main transient keys:

- ~@~ apply an options preset
- ~q~ open query submenu
- ~f~ open formatting submenu
- ~t~ run tail
- ~l~ run Logs Insights
- ~RET~ DWIM (tail if no query, Insights when query is set)
- ~-e~ toggle ECS JSON viewer for tail

* Package: json-log-viewer

~json-log-viewer~ is a generic, foldable JSON log viewer for Emacs.

It is callback-driven, keeps runtime state buffer-local, and supports:

- non-streaming mode (replace full dataset)
- streaming mode (append new lines)

** Quick API

- ~json-log-viewer-make-buffer BUFFER-NAME &key ...~ -> create/initialize a viewer buffer.
- ~json-log-viewer-get-buffer BUFFER-OR-NAME~ -> validate and return viewer buffer.
- ~json-log-viewer-push BUFFER-OR-NAME LOG-LINES~ -> append JSON lines in streaming mode.
- ~json-log-viewer-current-log-lines BUFFER-OR-NAME~ -> get current raw log lines.
- ~json-log-viewer-replace-log-lines BUFFER-OR-NAME LOG-LINES &optional PRESERVE-FILTER~ -> replace full dataset.

** Constructor

#+begin_src emacs-lisp
(json-log-viewer-make-buffer
 "*My Logs*"
 :log-lines '("{\"timestamp\":\"2026-02-17T10:00:00Z\",\"level\":\"info\",\"msg\":\"started\"}")
 :timestamp-path "timestamp"
 :level-path "level"
 :message-path "msg"
 :extra-paths '("meta.service" "meta.env")
 :json-paths '("payload" "payload.log")
 :refresh-function (lambda (old-log-lines) old-log-lines)
 :streaming nil
 :direction 'newest-first
 :header-lines-function (lambda (_state) (list (cons "Source" "demo"))))
#+end_src

Notes:

- ~:log-lines~ is a list of JSON strings.
- ~:refresh-function~ receives old raw lines and may return:
  - a new full list of JSON strings (synchronous refresh), or
  - the symbol ~:async~ (caller will update later via ~json-log-viewer-replace-log-lines~ or ~json-log-viewer-push~).
- ~:json-paths~ renders matching detail paths as pretty JSON blocks with syntax highlighting
  and suppresses flattened descendant fields for those paths.
- JSON block highlighting uses ~json-log-viewer-json-syntax-mode~ (default
  ~json-ts-mode~, with fallback to ~js-mode~ if unavailable).
- ~:streaming~ controls append semantics:
  - ~nil~: non-streaming order follows ~:direction~.
  - non-~nil~: pushes append to bottom.
- ~:header-lines-function~ contributes context lines shown in the ~?~ info popup.
- The package does not call ~display-buffer~; caller controls window behavior.

** Example: non-streaming

#+begin_src emacs-lisp
(let* ((all-lines
        '("{\"timestamp\":\"2026-02-17T10:00:00Z\",\"level\":\"info\",\"msg\":\"a\"}"
          "{\"timestamp\":\"2026-02-17T10:00:01Z\",\"level\":\"warn\",\"msg\":\"b\"}"))
       (buf
        (json-log-viewer-make-buffer
         "*JSON Logs (Non-Streaming)*"
         :log-lines all-lines
         :timestamp-path "timestamp"
         :level-path "level"
         :message-path "msg"
         :streaming nil
         :direction 'newest-first
         :refresh-function
         (lambda (_old-lines)
           (setq all-lines
                 (append all-lines
                         '("{\"timestamp\":\"2026-02-17T10:00:02Z\",\"level\":\"info\",\"msg\":\"c\"}")))
           all-lines)
         :header-lines-function
         (lambda (_state)
           (list (cons "Mode" "non-streaming demo"))))))
  (display-buffer buf))
#+end_src

** Example: streaming

#+begin_src emacs-lisp
(let ((buf
       (json-log-viewer-make-buffer
        "*JSON Logs (Streaming)*"
        :log-lines '("{\"timestamp\":\"2026-02-17T10:00:00Z\",\"level\":\"info\",\"msg\":\"first\"}")
        :timestamp-path "timestamp"
        :level-path "level"
        :message-path "msg"
        :streaming t
        :header-lines-function
        (lambda (_state)
          (list (cons "Mode" "streaming demo"))))))
  (display-buffer buf)
  (json-log-viewer-push
   buf
   '("{\"timestamp\":\"2026-02-17T10:00:01Z\",\"level\":\"warn\",\"msg\":\"second\"}"
     "{\"timestamp\":\"2026-02-17T10:00:02Z\",\"level\":\"info\",\"msg\":\"third\"}")))
#+end_src

** Keybindings

Inside ~json-log-viewer-mode~:

- ~TAB~: toggle current entry
- ~S-TAB~: toggle all entries
- ~C-c C-r~: refresh (when ~:refresh-function~ is configured)
- ~C-c C-n~: narrow by substring
- ~C-c C-w~: clear narrowing
- ~C-c C-f~: toggle auto-follow (auto-disables when cursor moves)
- ~?~: show viewer info popup
- ~q~: quit window

When Evil is installed and ~json-log-viewer-enable-evil-bindings~ is non-nil
(default), ~json-log-viewer.el~ will conditionally load
~json-log-viewer-evil.el~ and call ~json-log-viewer-setup-evil~. This adds:

- ~za~: toggle current entry
- ~zA~: toggle all entries
- ~zn~: narrow by substring
- ~zN~: clear narrowing
- ~gR~: refresh
- ~F~: toggle auto-follow

When Evil integration is active, these bindings replace the default list shown
in the ~?~ info popup.

* Package: kube-logs

~kube-logs~ provides a transient UI for ~kubectl logs~, rendered in ~json-log-viewer~.

** Features

- Switch context.
- Set namespace for log commands.
- Select target kind (~pod~ or ~deployment~) and target name.
- Toggle follow mode.
- Configure log limits with ~--tail~ and/or ~--since~.
- Create and apply named presets (similar to ~aws-logs~ presets).
- Render logs in ~json-log-viewer~ for fold/narrow/filter workflows.

** File

- ~kube-logs.el~

** Installation

Put ~kube-logs.el~ and ~json-log-viewer.el~ on your ~load-path~.

** use-package example

#+begin_src emacs-lisp
(use-package kube-logs
  :load-path "~/dev/emacs/aws-logs.el"
  :commands (kube-logs)
  :custom
  (kube-logs-kubectl "kubectl")
  (kube-logs-default-context "dev-cluster")
  (kube-logs-default-namespace "payments")
  (kube-logs-default-target-kind "deployment")
  (kube-logs-default-namespace-enabled t)
  (kube-logs-default-follow t)
  (kube-logs-default-tail-lines 300)
  :config
  (kube-logs-make-preset
   "prod-api"
   :context "prod-cluster"
   :namespace "payments"
   :namespace-enabled t
   :target-kind "deployment"
   :target "payments-api"
   :follow t
   :tail-lines 500
   :since "20m"))
#+end_src

** Presets

~kube-logs-make-preset~ stores named presets in ~kube-logs-presets~.

Supported keys:

- ~:context~
- ~:namespace~
- ~:namespace-enabled~ (non-nil means pass explicit namespace)
- ~:target-kind~ (\"pod\" or \"deployment\")
- ~:target~
- ~:follow~
- ~:tail-lines~
- ~:since~

Apply behavior:

- Key missing: keep current value.
- Key present with ~nil~: clear value.
- Key present with non-~nil~: set value.

** Usage

Run ~M-x kube-logs~.

Main transient keys:

- ~@~ apply preset
- ~c~ context
- ~n~ set namespace (also enables namespace override)
- ~-n~ set namespace (same behavior as ~n~)
- ~p~ select pod target
- ~d~ select deployment target
- ~-m~ tail lines
- ~-s~ since
- ~-f~ toggle follow
- ~RET~ run logs

* Package: kafka-logs

~kafka-logs~ provides a transient UI for Kafka message consumption via ~kcat~,
rendered in ~json-log-viewer~.

** Features

- Select a preconfigured connection.
- Resolve credentials through ~auth-source~.
- Select topics from cluster metadata.
- Stream new messages from topic end.
- Look up messages in a FROM/TO time span.
- Optional regex filtering with ~grep~.
- Optional payload decoding mode for JSON message bodies.
- Configurable summary message JSON path (including nested paths).
- Configurable summary extra JSON paths.
- Optional JSON-path detail rendering (for example ~payload~ as pretty JSON block).
- SQLite-backed hidden-data storage to reduce in-memory growth.

** File

- ~kafka-logs.el~

** Installation

Put ~kafka-logs.el~ and ~json-log-viewer.el~ on your ~load-path~.

** use-package example

#+begin_src emacs-lisp
(use-package kafka-logs
  :load-path "~/dev/emacs/aws-logs.el"
  :commands (kafka-logs)
  :custom
  (kafka-logs-kcat "kcat")
  (kafka-logs-default-connection "prod")
  (kafka-logs-default-topic "orders.events")
  (kafka-logs-default-stream t)
  (kafka-logs-default-message-path "payload.data.name")
  (kafka-logs-default-extra-paths '("connection" "topic" "partition"))
  (kafka-logs-default-payload-format 'json)
  (kafka-logs-default-json-paths '("payload"))
  :config
  ;; auth-source lookup (example entry source):
  ;; machine kafka-prod login app-user password s3cr3t port 9093
  (kafka-logs-make-connection
   "prod"
   :brokers "kafka-prod:9093"
   :security-protocol "SASL_SSL"
   :sasl-mechanisms "SCRAM-SHA-512"
   :auth-source t
   :properties '(("client.id" . "emacs-kafka-logs"))))
#+end_src

You can provide explicit auth-source query settings with ~:auth-source~ plist:

#+begin_src emacs-lisp
(kafka-logs-make-connection
 "staging"
 :brokers "kafka-staging:9093"
 :security-protocol "SASL_SSL"
 :sasl-mechanisms "PLAIN"
 :auth-source '(:host "kafka-staging" :port 9093 :user "staging-user"))
#+end_src

** Usage

Run ~M-x kafka-logs~.

Main transient keys:

- ~c~ select connection
- ~t~ select topic
- ~-f~ toggle stream/time-span mode
- ~a~ set FROM time (time-span mode)
- ~z~ set TO time (time-span mode)
- ~-F~ set regex filter
- ~-M~ set summary message path (for example ~payload~, ~payload.data.name~)
- ~-j~ set json detail paths (comma-separated, e.g. ~payload,payload.log~)
- ~-p~ toggle payload format (string/json)
- ~-m~ set max message count for time-span mode
- ~RET~ run logs

When the selected message path resolves to a JSON object or array, it is
rendered as compact one-line JSON in the summary line.
